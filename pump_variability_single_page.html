Multi-Pump Flow Variability Tool

Generic pump-channel flow, topology assumptions, and installed-system uncertainty

How to use this

Use the page to make pump-channel assumptions, nominal flow, and missing measurements visible.

  1. Start in System Model. Set the number of channels, assign each enabled pump a generic role, and use the abstract topology to check whether the model matches the conversation.
  2. Edit the Pump channel matrix. For each channel, choose pumphead, tube bore, wall, rpm, installation condition, and any one-run measured flow you have.
  3. Click a channel row or topology card to inspect it in Selected Pump Detail. Use this for single-channel nominal flow, mL/rev, granularity, and range warnings.
  4. Review System Results for gross pumped flow, process inflow/outflow, nominal balance error, recirculation flow, measurement coverage, unavailable rows, and over-limit rows.
  5. Use Scenario Variability only as a discussion layer. Toggle placeholder factors to see how assumptions could move the system band, then identify which assumptions need measurement.
  6. Use Measurement Quick Entry to add measured flow for a channel and see the one-run ratio, bias, and correction factor.
  7. Use Source / Data Panel to inspect the embedded vendor tables and the generic role definitions behind the system model.
This page is a conversation aid. Treat vendor nominal data, placeholder scenario factors, and one-run local measurements as separate evidence levels.
